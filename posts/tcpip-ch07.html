为何需要半关闭?

半关闭允许发送方停止发送数据，同时仍然接收来自对方的数据，这在需要确认或处理剩余数据的场景中非常有用。

shutdown

作用::shutdown 用于套接字的半关闭，所谓半关闭是指关闭传输流的一半，只关闭输入/输出，只保留输出/输入功能，例如，调用 shutdown(sock, SHUT_RD) 将关闭套接字的接收功能，但允许继续发送数据。

用法：

  • sock：文件描述符

  • howto：要断开的方式

    | 参数 | 作用 |
    | :———— | :———- |
    | SHUT_RD | 断开输入流 ，不发 FIN |
    | SHUT_WR | 断开输出流，发送 FIN |
    | SHUT_RDWR | 同时断开I/O流，发送 FIN |

注意：

  1. 使用 SHUT_RD 断开输入流后，会清空输入缓存数据，页无法调用五日相关的函数。使用 SHUT_WR 断开输出流后，如果输出缓存还存有未发送出去的数据，数据还会继续发送而不是清空
  2. 通过 shutdown(sock, SHUT_WR) 与 close 函数都会发送 FIN，表示要关闭套接字，对面的套接字 read 就会返回 EOF=0，表示：对方的写端已关闭，这条连接上永远不会再有新数据来了
  3. FIN 是 TCP 层的语言，EOF 是应用层的语言，它们说的是同一件事，只是层级不同。
    1. TCP 协议栈收到 FIN 标志位：对端关闭了写端
    2. 应用层 read() 返回 0：EOF（文件结束）
