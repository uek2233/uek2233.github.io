场景 命令
提交到了错误分支 git reset HEAD~1 → git stash → 切到正确分支 → git stash pop
误删文件 git checkout -- <file> 或 git restore <file>
已经 push 了错误提交 git revert <hash> 然后 push
rebase 冲突搞乱了 git rebase --abort
想取消 merge git merge --abort（未完成时）或 git reset --hard HEAD~1（已完成时）
本地改乱了想重置 git reset --hard origin/<branch>

十、推荐团队规范

  • 分支命名：feature/xxx bugfix/xxx hotfix/xxx release/x.x.x
  • 提交信息：Conventional Commits 格式
    • feat: add login page
    • fix: resolve null pointer in user service
    • refactor: extract common utils
    • docs: update API documentation
  • 合并策略：feature 分支用 rebase 保持线性历史；release/long-lived 分支用 merge --no-ff
  • 永远不要 push --force 到 main / master 分支
