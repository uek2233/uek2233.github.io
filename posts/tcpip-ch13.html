send&recv 函数

函数用法

☀️

作用:: 数据 I/O 读写

用法：

  • sockfd：表示数据传输对象连接的套接字文件描述符
  • buf：保存待传输数据的缓冲地址值
  • nbytes： 待传输的字节数

  • flags： 传输数据时指定的可选项信息

    |可选项(option)|含义|send|recv|
    |—-|—-|—-|—-|
    |MSG_OOB|用于传输带外数据(Out-of-band data)|Y|Y|
    |MSG_PEEK|验证输入缓冲是否存在接收的数据|N|Y|
    |MSG_DONTROUTE|传输传输过程中不参照路由(Routing)表，在本地(Local)网络中寻找目的地|Y|N|
    |MSG_DONTWAIT|调用I/O函数时不阻塞，用于使用非阻塞(Non-blocking)I/O|Y|Y|
    |MSG_WAITALL|防止函数返回，直到接收全部请求的字节数|N|Y|

MSG_OOB 发送紧急消息

MSG_OOB 并不是让 TCP 不按序传数据，而是在有序的字节流里“高亮”了一个字节，并通知接收方”这个字节很重要，赶紧看”。 TCP 的字节顺序纹丝不动，动的是接收方读取的优先级。

MSG_PEEK|MSG_DONTWAIT 检查输入缓冲

同时设置 MSG_PEEK 选项和 MSG_DONTWAIT 选项，以验证输入缓冲中是否存在接收的数据。MSG_PEEK 选项，即使读取了缓冲中的数据也不会删除，加上额外的选项 MSG_DONTWAIT，以非阻塞方式验证。

readv & writev 函数

比较项目 read/write readv/writev
函数原型 ssize_t read(int fd, void *buf, size_t count);

ssize_t write(int fd, const void *buf, size_t count);		 ssize_t readv(int fd, const struct iovec *iov, int iovcnt);

ssize_t writev(int fd, const struct iovec *iov, int iovcnt);
缓冲区操作 每次只能操作一个连续的缓冲区 可在一次系统调用中操作多个非连续的缓冲区
数据传输特点 适合简单、连续的数据传输 适用于数据需分散读取或集中写入的场景
系统调用次数 每次读写操作对应一次系统调用 一次系统调用可完成对多个缓冲区的读写
适用场景 简单数据结构的读写，对性能要求不高的场景 网络协议处理、分散 - 聚集 I/O，需减少系统调用次数的高并发场景
代码复杂度 相对简单，只需处理单个缓冲区 相对复杂，需管理 iovec 结构体数组

函数用法

☀️

作用:: 对数据进行整合传输及发送的函数，通过 writev 函数可以将分散保存在多个缓冲中的数据一并发送，通过 readv 函数可以由多个缓冲分别接收。

用法：

  • filedes：文件描述符，可以是套接字，也可以是普通文件描述符
  • iov：iovec 结构体数据的地址值，结构体iovec中包含待发送数据的位置和大小信息
  • iovcnt：iovec 结构体的数量

注意：

  1. 如果需要保存的数据大于第一个 iovec 结构体设置数据的大小，数据会保存到第二个结构体指定的保存位置
