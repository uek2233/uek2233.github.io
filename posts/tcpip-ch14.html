第14章 多播与广播

发表于2026-05-18|更新于2026-06-05|TCP-IP网络编程
|字数总计:1.2k|阅读时长:6分钟|阅读量:

总结

小结:: 本章主要介绍了多播与广播的概念以及实现方法，多播与广播都是基于 UDP 实现的

  1. 单播：一对一通信，使用 A/B/C 类 IP，是前面学习的 TCP/UDP 常规通信模式。
  2. 多播：通过 IP 地址来标识多播，依托 D 类 IP（224.0.0.0~239.255.255.255）组建多播组，可跨网段传输
    1. 多播服务器向特定的多播组( D 类 IP )发送只一次数据，客户端通过 setsockopt(IPPROTO_IP, IP_ADD_MEMBERSHIP) 函数来接入多播组，组内的客户端都会接到数据
    2. 多播依赖路由器转发报文，路由器会复制数据包传递到多个主机，每经过一个路由器 TTL(生存时间) -1
    3. 多播 sender 流程： socket👉setsockopt(IPPROTO_IP, IP_MULTICAST_TTL)👉sendto
    4. 多播 receiver 流程：socket👉bind👉setsockopt(IPPROTO_IP, IP_ADD_MEMBERSHIP)👉recvfrom
    5. 多播主要用于多媒体数据的实时传输场景
  3. 广播：仅局域网内生效，无法跨路由，广播分为直接广播与本地广播两种，通过 IP 地址区分
    1. 直接广播：网络地址外，主机地址为 255，例如 C 类地址是 192.168.0.x，192.168.0 是网络地址，x 是主机地址，那么直接广播的地址为 192.168.0.255，发往该地址的数据会被192.168.0 网络内的全部主机接收
    2. 本地广播：固定为 255.255.255.255
    3. 广播 sender 流程： socket👉setsockopt(SOL_SOCKET, SO_BROADCAST)👉sendto
    4. 广播 receiver 流程：socket👉bind👉recvfrom
