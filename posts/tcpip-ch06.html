第6章 基于UDP的服务器端与客户端

发表于2026-05-18|更新于2026-06-05|TCP-IP网络编程
|字数总计:1.4k|阅读时长:6分钟|阅读量:

总结

小结:: 本章讲解 UDP 套接字原理、特性、读写函数，以及已连接 / 未连接两种使用模式。

  1. UDP 无连接，传输效率更高，适合频繁交互、小数据量场景；存在数据边界，sendto 会附带地址信息。
  2. UDP 套接字流程
    1. 服务端：socket 👉bind 👉recvfrom👉sendto
    2. 客户端(已连接)：socket👉connect👉sendto
    3. 客户端(未连接)：socket👉sendto

UDP 套接字

UDP 与 TCP 对比

传输效率：

  • TCP 套接字中存在 1. 收发数据前后后三次握手，四次挥手的过程，2. 收发数据中有流控制的应答机制，这特点使得 TCP 开销更大
  • UDP 无连接流程，在频繁建连、小数据传输场景下效率更高。

连接过程：

  • TCP： 面向连接，需要使用 listen 与 accept 函数来建立监听套接字与客户端端的套接字
  • UDP： 无真正连接，每次收发可独立指定目标地址

数据边界：

  • TCP：字节流，没有数据边界，可以使用多次 write 然后使用 一次 read 来接收全部数据
  • UDP：有数据边界，使用多少次 write 就得使用 多少次 read 来接收数据

地址分配：

  • TCP：服务端靠 bind、客户端靠 connect 确定通信地址
  • UDP：无严格服务端 / 客户端区分，地址分配是通过 sendto 或者 connect 来实现，执行一次 sendto 都将进行注册 IP 与端口号，传输数据，注销 IP 与端口号的过程
    • 接收数据：bind + recvfrom 函数
    • 发送数据(未连接的 UDP)：直接使用 sendto 函数
    • 发送数据(已连接的 UDP)： connect + sendto 函数

sendto&recvfrom

☀️

作用:: udp 套接字的 I/O 函数

用法：

  • sock：传输数据的 UDP 套接字文件描述符
  • buf：保存待传数据的缓冲区地址
  • nbytes：传输的字节数
  • flags：可选项参数，若无则传 0
  • to/from：目标地址信息的结构体变量的地址值
  • addrlen：目标地址信息的结构体变量大小

注意：

  1. sendto 之前没有进行地址分配，sendto 会先分配地址，然后数据传输，最后回收地址

已连接的 UDP 套接字

UDP 是在发送数据的时候给套接字临时绑定IP地址和端口号，数据传输完成就解除绑定。这样的特性，可以让主机用同一个套接字给不同的主机发送数据。同样，在UDP在接收数据的时候给套接字临时绑定IP地址和端口号，这样就可以从不同的主机接收数据。提前分配地址信息的套接字称为已连接套接字，反之则为未连接套接字。
